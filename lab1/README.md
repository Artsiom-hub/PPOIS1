Система издания и распространения научных работ
Описание

Программная система предназначена для моделирования процессов издания и распространения научных работ и журналов.

Система реализует основные этапы издательского процесса:

прием научных статей

рецензирование статей

подготовка статей к публикации (верстка)

формирование журналов

выпуск печатных экземпляров

публикация электронной версии

распространение выпусков

Программа реализована на языке Python и работает через интерфейс командной строки (CLI).

Архитектура системы

Система построена по принципу разделения на несколько уровней:

CLI (main.py)
     ↓
Service layer (services.py)
     ↓
Domain entities (classes/)
Уровни архитектуры
CLI уровень

Файл:

Отвечает за:

взаимодействие с пользователем

вызов операций системы

Сервисный уровень

Файл:

services.py

Содержит бизнес-логику операций системы:

прием статей

рецензирование

верстка

выпуск электронной версии

Уровень сущностей

Каталог:

classes/

Содержит классы предметной области.

Основные сущности системы
Автор (Author)

Представляет автора научной статьи.

Атрибуты

id — уникальный идентификатор автора

name — имя автора

affiliation — организация

email — контактный email

Редактор (Editor)

Редактор выполняет рецензирование научных статей.

Атрибуты

id

name

field — область научной специализации

Методы

review_article()
Научная статья (ScientificArticle)

Представляет научную публикацию.

Атрибуты

id

title

authors

abstract

status

Статусы статьи

submitted
reviewed
layout_ready
Журнал (Journal)

Журнал содержит список опубликованных статей.

Атрибуты

id

name

field

articles

Методы

add_article()
Издательство (PublishingHouse)

Издательство управляет журналами.

Атрибуты

name

journals

Методы

add_journal()
Печатное издание (PrintedEdition)

Представляет печатный выпуск журнала.

Атрибуты

journal_id

issue_number

copies

articles

Методы

print_issue()
Электронная версия (ElectronicVersion)

Представляет электронную публикацию журнала.

Атрибуты

journal_id

url

Методы

publish()
Реализованные операции системы
Прием статьи

Добавление новой научной статьи в систему.

Используется сервис:

ArticleSubmissionService
Рецензирование

Рецензирование выполняется редактором.

Используется сервис:

PeerReviewService
Верстка

Подготовка статьи к публикации.

Используется сервис:

LayoutServiceImpl
Печать журнала

Формирование печатного выпуска журнала.

Используется сервис:

PrintService
Выпуск электронной версии

Публикация электронной версии журнала.

Используется сервис:

ElectronicPublishingService
Распространение

Распространение выпущенного журнала.

Используется сервис:

DistributionServiceImpl
Обработка ошибок

В системе используется механизм исключений Python.

Реализованы пользовательские исключения:

AuthorNotFoundError
EditorNotFoundError
ArticleNotFoundError
JournalNotFoundError
InvalidInputError

Это позволяет корректно обрабатывать ошибочные ситуации:

ввод некорректных данных

обращение к несуществующим объектам

ошибки пользовательского ввода

Сохранение состояния системы

Система сохраняет состояние между запусками программы.

Используется файл:

data/state.json

Сохраняются следующие данные:

авторы

редакторы

статьи

журналы

издательство

Загрузка и сохранение реализованы в модуле:

storage.py
Интерфейс командной строки

После запуска программы пользователю доступно меню.
