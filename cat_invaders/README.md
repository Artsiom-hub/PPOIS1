# Cat Invaders (Python + Pygame)

## 1. Общие сведения

Данное приложение представляет собой игровую программу в жанре аркадного шутера, реализованную на языке Python с использованием библиотеки Pygame. Игрок управляет космическим кораблём, уничтожает волны противников, собирает улучшения и стремится набрать максимальное количество очков.

Приложение разработано с применением событийно-ориентированной архитектуры и использует внешние конфигурационные файлы для хранения параметров игры.

---

## 2. Функциональные возможности

В рамках проекта реализованы следующие функции:

- отображение графического интерфейса и игрового поля;
- управление игровым персонажем с клавиатуры;
- генерация и прохождение не менее 20 волн противников;
- использование различных типов врагов с уникальным поведением;
- реализация нескольких типов оружия;
- система выпадения и подбора игровых предметов;
- обработка столкновений и расчёт урона;
- система подсчёта очков;
- отображение таблицы рекордов с сохранением результатов;
- наличие справочной информации по правилам игры;
- воспроизведение звуковых эффектов и фоновой музыки;
- реализация анимаций (взрывы, поведение объектов, полёт снарядов).

---

## 3. Архитектура приложения

Проект построен по модульному принципу с разделением на логические подсистемы.

### 3.1 Основные модули

- **core** — базовая логика приложения:
  - `game.py` — основной игровой цикл;
  - `scene_manager.py` — управление сценами;
  - `event_bus.py` — система обработки событий;
  - `resource_manager.py` — загрузка и хранение ресурсов;
  - `config_loader.py` — загрузка конфигурационных файлов.

- **entities** — игровые объекты:
  - игрок, противники, снаряды, эффекты, предметы.

- **systems** — системы обработки логики:
  - искусственный интеллект противников;
  - система боя;
  - система оружия;
  - система выпадения предметов;
  - система анимации;
  - система подсчёта очков.

- **scenes** — сцены приложения:
  - главное меню;
  - игровой процесс;
  - пауза;
  - экран поражения;
  - справка;
  - таблица рекордов.

---

## 4. Событийно-ориентированная модель

В приложении реализована собственная система событий (EventBus), обеспечивающая взаимодействие между компонентами.

Основные принципы:

- компоненты не взаимодействуют напрямую;
- логика приложения построена на генерации и обработке событий;
- используется подписка на события и их обработка соответствующими системами.

Пример цепочки событий:

- пользователь нажимает клавишу стрельбы;
- генерируется событие `player_shot_requested`;
- система оружия создаёт снаряд;
- система столкновений обрабатывает попадание;
- система очков увеличивает счёт игрока.

Данный подход снижает связанность компонентов и упрощает расширение функциональности.

---

## 5. Конфигурация приложения

Все основные параметры игры вынесены во внешние файлы в формате JSON.

Структура каталога конфигураций:

- `game.json` — общие настройки (окно, управление, звук);
- `player.json` — параметры игрока;
- `enemies.json` — характеристики противников;
- `weapons.json` — параметры оружия;
- `drops.json` — описание выпадающих предметов;
- `ai.json` — параметры поведения искусственного интеллекта;
- `waves/` — описание волн противников;
- `high_scores.json` — таблица рекордов.

Использование конфигурационных файлов позволяет изменять параметры игры без изменения исходного кода.

---

## 6. Анимации

В приложении реализованы анимации следующих элементов:

- анимация уничтожения объектов (взрывы);
- визуальные изменения снарядов в процессе движения;
- анимация поведения противников (изменение формы и положения).

Анимации реализованы как последовательность кадров или процедурные изменения графических объектов.

---

## 7. Звуковое сопровождение

Приложение использует библиотеку `pygame.mixer` для воспроизведения звука.

Реализованы:

- фоновая музыка (меню, игровой процесс, завершение игры);
- звуки выстрелов;
- звуки взрывов;
- звуки подбора предметов.

Пути к аудиофайлам задаются в конфигурационных файлах.

---

## 8. Таблица рекордов

Результаты игроков сохраняются в файл `high_scores.json`.

При завершении игры выполняется проверка результата:
- если результат превышает лучший, отображается форма ввода имени;
- введённое имя и результат сохраняются в таблицу рекордов.

---

## 9. Управление

Основные клавиши управления:

- движение — стрелки влево и вправо либо клавиши A и D;
- стрельба — клавиша SPACE;
- пауза — клавиша ESC.
