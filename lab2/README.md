Отчёт по разработанной системе
Назначение системы

Разработано настольное приложение для учёта и анализа данных о студентах и их общественной активности по семестрам. Система позволяет хранить, отображать, фильтровать, удалять и сохранять записи в различных форматах (база данных и XML).

Функциональные возможности

Система реализует следующий набор функций:

добавление записей о студентах;

отображение списка записей в табличном виде;

альтернативное отображение данных в виде дерева;

поиск записей по заданным условиям:

по фамилии;

по группе;

по диапазону значений активности;

удаление записей по аналогичным условиям;

отображение результатов удаления (количество удалённых записей);

постраничный вывод данных (пагинация);

изменение количества записей на странице;

сохранение данных:

в базе данных SQLite;

в XML-файл (через DOM);

загрузка данных из XML-файла (через SAX).

Структура данных

Каждая запись содержит:

ФИО студента (строка);

номер группы (строка);

значения общественной активности за 10 семестров (массив целых чисел).

Типы данных выбраны корректно:

строки — для текстовых полей;

список целых чисел — для активности;

в базе данных используется текстовое представление с последующим преобразованием.

Архитектура системы

Приложение реализовано по шаблону MVC:

Model

Student — сущность;

StudentStorage — работа с базой данных, фильтрация, удаление, загрузка/сохранение.

View

главное окно (MainWindow);

диалоги поиска и удаления;

таблица и дерево;

элементы управления пагинацией.

Controller

связывает UI и модель;

обрабатывает действия пользователя (поиск, удаление, загрузка и т.д.).

Хранение данных

Используются два способа хранения:

1. База данных (SQLite)

таблица students;

хранение всех записей;

используется для постоянного хранения данных.

2. XML-файлы

используются для импорта/экспорта данных;

запись — через DOM-парсер;

чтение — через SAX-парсер;

реализована загрузка через стандартный файловый диалог.

Отображение данных
Табличный режим

классический вид данных;

колонки: ФИО, группа, 10 семестров.

Древовидный режим

корень — студент;

дочерние элементы:

группа;

значения активности по семестрам.

Пагинация

Реализован постраничный вывод:

кнопки:

переход в начало;

назад;

вперёд;

в конец;

отображается:

текущая страница;

общее количество страниц;

общее количество записей;

можно менять количество записей на странице.

Сценарии демонстрации системы

Вот ключевая часть, за которую реально ставят оценку.

Сценарий 1 — Загрузка данных

Запустить приложение.

Нажать «Загрузить XML».

Выбрать файл students1.xml.

Результат:

данные отображаются в таблице;

обновляется пагинация.

Сценарий 2 — Просмотр данных

Просмотреть данные в таблице.

Переключиться в режим «Дерево».

Результат:

отображение меняется;

структура данных видна по полям.

Сценарий 3 — Пагинация

Установить количество записей на странице (например, 5).

Переключаться кнопками:

вперёд;

назад;

в конец.

Результат:

страницы корректно переключаются;

отображается номер текущей страницы.

Сценарий 4 — Поиск

Открыть диалог поиска.

Ввести:

фамилию (например, «Иванов»);

или диапазон активности (например, 5–10).

Нажать «Найти».

Результат:

отображаются только подходящие записи;

работает совместно с пагинацией.

Сценарий 5 — Удаление

Открыть диалог удаления.

Задать условия (например, группа + диапазон).

Нажать «Удалить».

Результат:

записи удаляются из базы;

отображается сообщение:

сколько записей удалено;

либо что ничего не найдено.

Сценарий 6 — Сохранение

Нажать «Сохранить XML».

Указать путь.

Результат:

создаётся XML-файл с текущими данными.

Сценарий 7 — Работа с несколькими файлами

Последовательно загрузить:

students1.xml

students2.xml

students3.xml

Результат:

система корректно обрабатывает разные наборы данных;

каждый файл содержит ≥50 записей.

Итог

Разработанная система:

реализует полный CRUD-функционал;

поддерживает два формата хранения данных;

обеспечивает удобную работу с большими объёмами данных за счёт пагинации;

предоставляет альтернативные способы визуализации;

соответствует требованиям задания.
